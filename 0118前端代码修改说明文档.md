## 前端代码修改说明文档
修改内容总览

### 1. 页面结构调整
新增了一个中间页面：video.html（视频管理页面）
原有结构：home.html → tasks.html
新的结构：home.html → video.html → tasks.html
### 2. 导航流程变更
新的用户操作流程：
2.1 登录后进入首页（home.html）
2.2 点击俱乐部卡片的"查看任务"按钮 → 跳转到video.html
2.3 在video.html创建任务 → 显示任务列表
2.4 点击任务卡片 → 跳转到tasks.html完成任务

### 3. 详细修改说明
3.1 新增页面：video.html
位置：pages/video.html
功能：任务创建与管理页面
主要组件：
o顶部导航栏（返回按钮、俱乐部名称、创建任务按钮）
o欢迎区域（显示当前俱乐部信息）
o任务列表区域（显示已创建的任务）
o创建任务模态框
3.2 修改文件：clubs.js
修改位置：renderClubList()函数中的俱乐部卡片按钮
// 修改前：
onclick="goToVideoPage(${club.id})"
// 修改后：
onclick="window.App.enterVideoPage && window.App.enterVideoPage(${club.id})"
3.3 修改文件：app.js
新增功能：进入视频页面
// 新增函数：
enterVideoPage: function(clubId) {
// 设置当前俱乐部
if (window.Clubs && window.Clubs.setCurrentClub) {
window.Clubs.setCurrentClub(clubId);
}
// 更新App状态
this.state.currentClubId = clubId;
// 跳转到视频页面
this.navigateTo('video');
}
3.4 修改文件：config.js
新增配置：
// 在PAGE_PATHS中添加video页面
PAGE_PATHS: {
// ... 其他页面
VIDEO: 'pages/video.html', // 新增
TASKS: 'pages/tasks.html'
}
3.5 任务页布局优化：tasks.html
主要改进：
1.简化了任务显示逻辑：所有任务页面显示相同的两个固定任务
2.移除了复杂的参数传递：不再需要通过URL传递任务ID
3.优化了用户界面：
o清晰的顶部导航（返回按钮）
o两个任务卡片并排显示
o每个任务包含：任务标题、状态、描述、平台信息和操作按钮
3.6 任务创建流程简化
video.html中的创建任务功能：
只需填写任务名称，无需其他复杂信息
创建后自动生成两个固定子任务：
1.看视频任务（跳转分秒帧平台）
2.研视频任务（跳转石墨文档）

### 4. 代码结构变化
修改前：
├── home.html # 首页
└── tasks.html # 任务页
修改后：
├── home.html # 首页
├── video.html # 新增：视频管理页
└── tasks.html # 任务页（优化布局）
### 5. 关键功能点说明
5.1 任务创建流程
点击"创建任务" → 弹出模态框 → 输入任务名称 → 确认
↓
创建成功 → 显示任务卡片 → 点击卡片 → 进入tasks.html
↓
显示两个固定任务 → 分别完成看视频和研视频任务
5.2 两个固定任务
无论创建什么任务，进入tasks.html都显示：
1.看视频任务
o平台：分秒帧
o链接：https://app.mediatrack.cn/projects/2009960971614818304/files
o完成标记：确认已观看视频
2.研视频任务
o平台：石墨文档
o链接：https://shimo.im/space/2wAldmGZonhwbwAP
o完成标记：确认已提交笔记
